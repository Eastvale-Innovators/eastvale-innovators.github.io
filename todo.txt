# Eastvale Innovators Pre-Publish Master Checklist

Use this as a complete launch gate. Do not publish until all P0 items are done and verified.

## P0 - Launch Blockers (Must Be Done Before Publish)

- [ ] Replace placeholder social links on Contact page (`href="#"`) with real URLs or remove buttons.
- [ ] Implement real contact form submission (serverless endpoint/service) instead of `alert()` only.
- [ ] Add `name` attributes to all form inputs/select/textarea so submissions are valid.
- [ ] Add form validation and failure/success states (inline errors, submit disabled while sending, retry state).
- [ ] Remove or hide any unfinished template pages from nav/internal links if not production-ready.
- [ ] Ensure no malformed HTML remains (especially from third-party embed snippets).
- [ ] Verify all navigation links resolve to valid pages/anchors (desktop + mobile menus).
- [ ] Fix or remove broken image references and placeholder profile images if real assets exist.
- [ ] Confirm all external assets load on production domain (CDN scripts, fonts, embeds, model files).
- [ ] Run cross-browser smoke tests: Chrome, Edge, Safari, Firefox (latest stable).
- [ ] Run responsive smoke tests: 320, 375, 390, 414, 768, 1024, 1280, 1440 widths.
- [ ] Ensure all pages render without console errors in production.
- [ ] Validate that shared header/footer loaders fail gracefully if partial fetch fails.
- [ ] Add no-JS fallback for critical navigation/footer content (or server-side include alternative).
- [ ] Remove any dead code that can trigger runtime issues if uncommented later.

## P1 - High Priority Code/Architecture

- [ ] Move large inline `<style>` blocks to external CSS files for cacheability and maintainability.
- [ ] Move large inline `<script>` blocks to external JS modules per page.
- [ ] De-duplicate tile transition logic repeated across multiple pages into one shared module.
- [ ] De-duplicate starfield/parallax patterns into reusable utilities.
- [ ] Centralize shared design tokens (colors/spacing/radii/shadows/timing) in one CSS variables file.
- [ ] Add code comments only where behavior is non-obvious (complex animation/math sections).
- [ ] Standardize script loading strategy (`defer` where appropriate).
- [ ] Add robust null checks around all DOM query selectors before use.
- [ ] Guard heavy effects behind `prefers-reduced-motion` and device capability checks.
- [ ] Add a basic error UI/fallback when third-party scripts fail to load.
- [ ] Reduce global event listeners where possible (especially high-frequency mousemove/scroll handlers).
- [ ] Add throttling/debouncing consistently for resize/scroll/mousemove handlers.
- [ ] Audit memory/perf for animation loops and avoid unnecessary per-frame allocations.
- [ ] Introduce a lightweight linting/formatting setup and run across HTML/CSS/JS.
- [ ] Create a small shared utilities file for repeated helper functions.

## P1 - Content Accuracy and Completeness

- [ ] Replace all placeholder/dummy profile images (`nil.jpg`) with final portraits or intentional branded placeholders.
- [ ] Verify all member names, roles, and bios for spelling and current accuracy.
- [ ] Verify all project names/descriptions/status labels are current.
- [ ] Ensure every CTA has a real destination and business purpose.
- [ ] Confirm contact email/account ownership and active inbox monitoring.
- [ ] Confirm legal ownership/permission for every image, icon, and third-party media embed.
- [ ] Review copy for consistent tone and grammar across all pages.
- [ ] Confirm leadership visibility decision (currently commented out in header nav).
- [ ] Decide whether `project-template.html` should ship publicly; if yes, fully sanitize placeholders first.

## P1 - Accessibility (WCAG-Focused)

- [ ] Ensure one clear `h1` per page and logical heading hierarchy.
- [ ] Add keyboard-only test pass for every interactive element.
- [ ] Ensure visible focus styles on links, buttons, form controls, and custom controls.
- [ ] Verify dropdown menus are fully keyboard-accessible (open/close, escape, focus return).
- [ ] Add/verify focus trap behavior for mobile menu dialog.
- [ ] Ensure all controls have accessible names (`aria-label` only where needed).
- [ ] Verify color contrast for text/buttons/cards against all backgrounds.
- [ ] Respect `prefers-reduced-motion` by reducing/paralyzing heavy animations.
- [ ] Ensure form validation messages are screen-reader friendly.
- [ ] Add landmark structure checks (`header`, `main`, `footer`, nav labeling consistency).
- [ ] Ensure decorative canvases/animations are hidden from assistive tech if non-semantic.
- [ ] Add skip-link for keyboard users.

## P1 - Performance and Loading

- [ ] Add preconnect hints for Google Fonts and any high-latency third-party domains.
- [ ] Audit and compress large images (AVIF/WebP + proper dimensions).
- [ ] Lazy-load all non-critical images and embeds.
- [ ] Add explicit width/height or aspect-ratio to media to reduce CLS.
- [ ] Replace heavy background effects on low-end devices with simpler fallbacks.
- [ ] Reduce JS payload and runtime on first load (especially home and projects pages).
- [ ] Add cache-busting strategy/versioning for static assets on deploy.
- [ ] Evaluate Three.js usage cost on home page and optimize quality/perf tradeoff.
- [ ] Verify canvas render loops pause when tab is hidden (or on reduced motion preference).
- [ ] Run Lighthouse and record baseline targets for Performance/Best Practices.

## P1 - SEO and Discoverability

- [ ] Verify every page has unique title/description and correct canonical URL.
- [ ] Ensure Open Graph/Twitter image URLs are valid and accessible.
- [ ] Add structured data JSON-LD (Organization, WebSite, potentially BreadcrumbList).
- [ ] Create and publish `sitemap.xml`.
- [ ] Create and publish `robots.txt`.
- [ ] Add favicon variants (`.ico`, apple-touch-icon, android icons) and manifest if needed.
- [ ] Validate social share previews for all key pages.
- [ ] Ensure internal linking strategy covers all key pages and sections.
- [ ] Submit sitemap to Google Search Console after launch.

## P1 - Security and Privacy

- [ ] Add `rel="noopener noreferrer"` to all `target="_blank"` links.
- [ ] Consider subresource integrity (SRI) for CDN scripts where feasible.
- [ ] Add CSP guidance via hosting config or meta fallback strategy.
- [ ] Validate no secrets/tokens/private data are present in source.
- [ ] Add abuse protection/rate limiting strategy for contact form endpoint.
- [ ] Add spam prevention (honeypot, CAPTCHA alternative, backend validation).
- [ ] Decide and document data retention policy for contact submissions.
- [ ] Add privacy notice if collecting personal information.

## P2 - Design and UX Polish

- [ ] Normalize section spacing rhythm across pages.
- [ ] Align card styles and hover intensities for visual consistency.
- [ ] Review typography scale consistency between hero, section titles, and body copy.
- [ ] Ensure button hierarchy is clear (primary vs secondary actions).
- [ ] Improve empty/loading/error states for dynamic components.
- [ ] Verify animation timing consistency and remove distracting motion.
- [ ] Ensure tile transition feels intentional and not blocking perceived responsiveness.
- [ ] Re-check footer visual consistency on every page after all content updates.
- [ ] Confirm nav dropdown behavior on touch devices is intuitive.
- [ ] Add subtle but consistent feedback for all interactions.

## P2 - QA Matrix and Regression Checks

- [ ] Build a page-by-page manual QA checklist and run it before each release.
- [ ] Add visual regression screenshots for key breakpoints.
- [ ] Test with network throttling (Fast 3G/Slow 4G) for first-load behavior.
- [ ] Test with CPU throttling for animation-heavy pages.
- [ ] Test no-mouse navigation flows end to end.
- [ ] Test with browser zoom at 125%, 150%, 200%.
- [ ] Validate contact form flow across desktop and mobile.
- [ ] Validate all in-page anchors scroll to correct sections.
- [ ] Validate all external links open correctly and are not dead.
- [ ] Confirm no overlap/cutoff on small-height mobile screens.

## P2 - GitHub Pages Deployment and Operations

- [ ] Confirm branch/source settings for GitHub Pages are correct.
- [ ] Confirm custom domain DNS records and HTTPS cert status.
- [ ] Verify `CNAME` matches intended canonical host strategy.
- [ ] Add/verify a custom `404.html` page.
- [ ] Add release checklist section to `Readme.md`.
- [ ] Document rollback plan if a bad deploy happens.
- [ ] Add changelog/release notes practice for public updates.

## P2 - Documentation and Team Workflow

- [ ] Document component/page ownership (who updates what).
- [ ] Document style and content conventions for future contributors.
- [ ] Document image asset naming and optimization workflow.
- [ ] Document how shared header/footer partial loading works.
- [ ] Document how to add new project cards/pages without duplicating logic.
- [ ] Add contributor instructions for testing before merge.

## Nice-To-Have (Post-Launch Enhancements)

- [ ] Add analytics event tracking for key interactions (CTA clicks, project views, form submits).
- [ ] Add uptime monitoring and error monitoring.
- [ ] Add newsletter or mailing list integration if needed.
- [ ] Add richer project detail pages and case studies.
- [ ] Add CMS/content pipeline if non-devs need to edit content regularly.
- [ ] Add localization readiness plan if multilingual support is expected.

## Final Launch Gate (Sign-Off)

- [ ] P0 complete and verified by manual QA.
- [ ] P1 complete or consciously deferred with owner + deadline.
- [ ] Stakeholder review done (content + branding + legal/privacy).
- [ ] Production deploy verified on live domain.
- [ ] Post-launch monitoring checklist active for first 72 hours.
